    Website Update and HEROBOTS Gets a New Logo

    July 10, 2020
    Website

    This site is developed using a static generator called Hugo and the Hugo template called Book created by Alex Shpak. It’s taking me some time to understand how to both use Hugo and get my head around the various theme components, configuration and how to structure the website.

    The way Hugo or any static generators work, is once you have all your templates and layouts setup, creating a new page or updating an existing page can simply be done in markdown within a file-based folder hiearchy structure. Markdown files are basically plain text files that are easy to edit and create so you don’t have to mess around with raw HTML/CSS inline (as much!). Once done, you run Hugo which applies all the HTML/CSS, links, and navigation structure and it generates all the static HTML files that you can simply push to any web hosting since it doesn’t require anything else to work - it doesn’t need a database or any server side applications. More on this later…

    I imagine I’ll be customizing the Book theme quite a bit, and adding my own “content types” and “archetypes” as I properly structure the various types of information for HEROBOTS. Hugo seems incredibly flexible, and once properly setup, easy to maintain and edit as I go along researching and compiling information for this site. Hopefully you’ll see more improvements to the site soon!

    So many people might wonder, why don’t you just use a standard content management system like Wordpress or any number of popular WYSIWIG site builder tools out there?

    A few reasons why, when you look at a single post or page on this site, it’s very straight forward - it’s just plain text! This makes it super easy to edit, doesn’t require a lot of “web designing” skills, and lets me focus on information itself. In fact, I’m typing this post in a basic text editor that I can do pretty much anywhere and pretty much on any computing device.

    Secondly, as I’ve seen through my research on the Heathkit HERO or other websites that were very clearly created in the past, a lot of them look super dated or do not render properly in new form factors (like mobile or table). But more importantly, in some cases, the authors have moved on in life and haven’t maintained the CMS platforms they were built on whether it be Wordpress or Frontpage or whatever tool it depends on including its database, applications or even programming language it requires to work.

    At best, this makes those sites hard to maintain over time and if you’ve moved onto a new project or phase of your life, no one wants to spend the extra time to update the underlying platform of a past project. At worst though, an unmaintained platform can be compromised by nefarious actors who takeover known exploits in Wordpress or popular CMS tools, or it may simply break over time and thereby taking down all the information within.

    In fact, some of the best information on HERO were from the old usenet newsgroups, list servers, and sometimes plain text that Google found. Because they were largely plain text, it stood up to the test of time and made it easy to search or grep through to find what I was looking for. See the following example of “plain text” file that Donnie Savage (who was super active with HERO robots in the past) preserved on his site, it’s also an excellent read! Hero Robot Frequently Asked Questions

    And finally, if properly done well, this site and the information I compile, should be easy for others to contribute or help maintain. I’ll explain more on that later when the site makes more progress. I’m still gathering as much information and source material about Heathkit HERO and consolidating it into one place so I can later figure out how to best represent my findings here on this site. Then, hopefully I’ll share methods or ways for contributors to help.
